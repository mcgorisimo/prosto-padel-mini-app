# QA_CHECKLIST.md

## Перед завершением задачи
- npm.cmd run test:e2e
- npm.cmd run build

## Основной MVP-flow
- регистрация работает
- создание открытого матча работает
- матч появляется в ленте
- детали открываются
- join занимает слот
- слот сохраняется после reload
- чат отправляет сообщение
- чат сохраняется после reload
- приватная бронь не попадает в публичную ленту
- нет ложных текстов оплаты

## Запрещено считать задачу готовой, если
- build падает
- e2e падает
- есть console.error
- есть Supabase 400/401/403/500
- пользователь видит “Оплатить”, когда реальной оплаты нет
