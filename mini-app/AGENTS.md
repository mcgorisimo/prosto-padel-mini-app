# AGENTS.md — правила работы для Codex

## Проект
Telegram Mini App для падел-клуба «Просто Падел».

Цель MVP:
игрок может зарегистрироваться, создать открытый матч, увидеть матч в ленте, присоединиться к матчу, занять слот, написать в чат, создать приватную бронь.

## Главный продуктовый сценарий MVP
1. Вход / регистрация.
2. Создание открытого матча.
3. Матч появляется в ленте.
4. Детали матча открываются.
5. Игрок присоединяется.
6. Слот сохраняется.
7. Чат работает.
8. Приватная бронь не попадает в публичную ленту.

## Строгие правила
- Не переписывать App.jsx целиком.
- Не менять дизайн без необходимости.
- Не менять Supabase-схему без отдельной команды.
- Не добавлять новые функции без отдельной команды.
- Не трогать payment-поля: paymentStatus, ownerPaid, holdAmount, prepay.
- Работать маленькими изменениями.
- Не делать длинные отчеты.
- Не пересказывать весь проект.
- Если задача непонятна — сначала посмотреть код, не задавать лишние вопросы.
- Если найден баг — исправить минимально.

## Формат работы
1. Прочитать TASK.md.
2. Изменить только нужные файлы.
3. Запустить:
   npm.cmd run test:e2e
   npm.cmd run build
4. Дать короткий отчет:
   - файлы изменены
   - что исправлено
   - test:e2e результат
   - build результат
   - что осталось

## Тесты
Smoke-тесты Playwright уже настроены.
Команды:
npm.cmd run test:e2e
npm.cmd run build

## Экономия лимита
- Не делать полный аудит проекта без запроса.
- Не читать все файлы подряд.
- Использовать точечный поиск rg.
- Не писать длинные объяснения.
- Не предлагать 10 следующих шагов.
